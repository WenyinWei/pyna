RMP Stellarator Resonance Analysis#

This notebook is the main public workflow for analytic stellarator resonance geometry. It folds two formerly separate text-only tutorials into one visual computation:

  1. Build an analytic stellarator equilibrium and trace Poincare sections.

  2. Compute resonant RMP Fourier components and their analytic X/O fixed points.

  3. Promote raw section points into geometry: crossings, fixed-point markers, resonant surfaces, O-point island-width bars, local stable branches, and coordinate overlays.

  4. Compare unperturbed and perturbed sections with a PEST-style grid.

  5. Summarize island widths, Chirikov overlap, and the \((m,n)\) spectrum.

  6. Produce a modern multi-section figure using pyna.plot helpers.

The notebook is designed to be executed locally before publishing the docs. GitHub Pages renders the saved outputs rather than recomputing field-line traces.

[EQ] Build the stellarator equilibrium#

We use an analytic single-helicity stellarator with:

  • Major radius \(R_0 = 3.0\) m, minor radius \(r_0 = 0.3\) m, on-axis field \(B_0 = 2.5\) T

  • Linear \(q\)-profile: \(q_0=1.5\) (axis) → \(q_1=4.5\) (LCFS)

  • Helical ripple: \((m_h, n_h) = (3,3)\), \(\epsilon_h = 0.03\)

The safety factor profile \(q(\psi) = q_0 + (q_1-q_0)\psi\) spans the range \([1.5, 4.5]\), so resonances at \(q = 2/1, 3/1, 4/1\) (among others) lie within the plasma.

[RMP_NRMP_WORKFLOW] RMP/nRMP decomposition#

A perturbation mode on a flux surface is useful only after we compare it with the local field-line rotation. With the Fourier convention

\[F(\theta,\varphi)=\sum_{m,n} F_{mn}\exp[i(m\theta+n\varphi)],\]

the local detuning is

\[\Delta_{mn}=m\iota+n, \qquad \iota=1/q.\]

The crucial difference is that an RMP diagnostic focuses on the resonant rows, while an nRMP response is a full sum over all non-resonant rows:

\[\delta r(\theta,\varphi)=\sum_{m\iota+n\ne 0}\frac{F_{r,mn}}{i(m\iota+n)}\exp[i(m\theta+n\varphi)].\]

Contribution tables are useful for ranking and convergence, but the model is the complete non-resonant spectrum, not a selected top component.

[RMP_FIELD] Define and visualise the RMP perturbation field#

We apply a single-mode RMP with base mode \((m,n)=(2,1)\) and amplitude \(\delta B=1\) mT. The user-facing template is

\[B_r = B_\mathrm{rmp}\cos(m\theta-n\varphi+\alpha),\]

but radial_rmp_field_template also adds the compensating poloidal/toroidal components needed to make the full cylindrical vector field divergence-free in the local circular-shell metric. The plotted radial projection is still the familiar resonant drive \(\delta B^r\).

The same helper supports the important m=1 branch; that case needs a toroidal component because the radial divergence contains a theta-independent part.

[POINCARE_PERTURBED] Geometry promotion: crossings -> X/O points -> manifolds#

The perturbed trace gives sampled Poincare points. The analytic RMP spectrum gives fixed-point and island-width predictions. pyna.plot.draw_rmp_resonance_section combines them into one section geometry:

  • Poincare points colored by flux label;

  • PEST-style \((S,\theta^*)\) grid lines;

  • resonant surfaces for each harmonic;

  • O-points and X-points from the analytic fixed-point formula;

  • O-point radial bars whose length is set by the resonant Fourier amplitude;

  • local stable separatrix branches born from the X-points.

This is the same promotion idea used by the general geometry workflow: raw samples are kept distinct from persistent geometric objects and overlays until an explicit model or diagnostic justifies the promotion.

    psi = eq.psi_ax(R, Z)

    r_minor = np.sqrt((R - R0_eq)**2 + Z**2)

    if r_minor > 1e-10:
        BR0 = -B_pol * np.sin(theta)
        BZ0 =  B_pol * np.cos(theta)
    else:
        BR0 = BZ0 = 0.0
    delta_BR_eq = eq.epsilon_h * eq.B0 * psi * np.cos(eq.m_h * theta - eq.n_h * phi)
    db = delta_B_RMP(R, Z, phi)
    BR_tot = BR0 + delta_BR_eq + db[0]
    BZ_tot = BZ0 + db[1]
    B_phi_tot = B_phi + db[2]
    B_mag = np.sqrt(BR_tot**2 + BZ_tot**2 + B_phi_tot**2) + 1e-30
    return np.array([BR_tot/B_mag, BZ_tot/B_mag, B_phi_tot/(R*B_mag)])

[NONRESONANT_DEFORMATION] Non-resonant ripple as flux-surface deformation#

The helical ripple in this analytic equilibrium is not the resonant RMP component that opens the (m,n)=(2,1) island. It still changes the geometry of the nearby flux surface. If we compare Newton fixed points against the undeformed circular surface, that smooth displacement appears as an artificial phase error.

Here we sample the helical-ripple contribution to the field-line ODE,

\[\frac{dr}{d\phi}=F_r(\theta,\phi), \qquad \frac{d\theta}{d\phi}=\iota+F_\theta(\theta,\phi),\]

Fourier transform F_r and F_theta, and solve the non-resonant homological equation for every non-resonant coefficient. The response object keeps the total deformation and a contribution ranking; the ranking is a diagnostic, while the deformation used below is the full summed nRMP response.

def helical_ripple_delta_B(eq_case):
    """Return the analytic helical-ripple contribution used by simple_stellarator."""
    def delta_B_helical(R, Z, phi):
        R_arr = np.asarray(R, dtype=float)
        Z_arr = np.asarray(Z, dtype=float)
        phi_arr = np.asarray(phi, dtype=float)
        theta = np.arctan2(Z_arr, R_arr - eq_case.R0)
        psi = eq_case.psi_ax(R_arr, Z_arr)
        dBR = eq_case.epsilon_h * eq_case.B0 * psi * np.cos(eq_case.m_h * theta - eq_case.n_h * phi_arr)
        return np.array([
            dBR,
            np.zeros_like(dBR, dtype=float),
            np.zeros_like(dBR, dtype=float),
        ])
    return delta_B_helical

[ORDER_ANALYSIS] Perturbation-order checks#

The reusable workflow now lives in a few small helpers: scan_nonresonant_residual_order, scan_rmp_amplitude_order, scan_rmp_phase_order, scan_rmp_resolution_convergence, and plot_perturbation_order_summary.

The expected orders are simple once the Fourier convention is fixed:

  • nRMP surface-shape residual: a first-order deformation should leave an O(k^2) map residual.

  • RMP resonant coefficient: for delta B = k f, Fourier linearity gives |b_{m,-n}| ~ k.

  • Island width: the Rutherford/Nardon pendulum width scales as w ~ sqrt(|b_{m,-n}|), hence w ~ k^{1/2}.

  • X/O phase: the phase is controlled by arg(b_{m,-n}), not by amplitude. The exact relation is m*Delta theta_O + Delta arg(b_{m,-n}) = 0.

For phase-control order we use the divergence-free template radial_rmp_field_template. Its phase parameter changes the resonant coefficient phase while preserving div(delta B)=0, including the important m=1 case. We deliberately test a mildly nonlinear control phase alpha(k)=k+eta*k^2; the residual against the first-order raw-k law should therefore scale as O(k^2).

def deformed_torus_map_residual(epsilon_h, n_alpha=12):
    eq_case = simple_stellarator(
        R0=eq.R0, r0=eq.r0, B0=eq.B0,
        q0=eq.q0, q1=eq.q1,
        m_h=eq.m_h, n_h=eq.n_h, epsilon_h=float(epsilon_h),
    )
    psi_res = eq_case.resonant_psi(base_m, base_n)[0]
    deformation, r_res, _ = helical_velocity_deformation(
        eq_case, psi_res, n_theta=128, n_phi=128, include_shear=True
    )
    iota = 1.0 / float(eq_case.q_of_psi(psi_res))

    def surface(alpha, phi):
        alpha_arr = np.asarray(alpha)
        return (
            r_res + deformation.section_r(alpha_arr, phi),
            alpha_arr + deformation.section_theta(alpha_arr, phi),
        )

    def rhs(phi, state):
        radius, theta = state
        R = eq_case.R0 + radius*np.cos(theta)
        psi_here = (radius / eq_case.r0)**2
        q_here = float(eq_case.q_of_psi(psi_here))
        Bphi = eq_case.B0 * eq_case.R0 / R
        delta_BR = eq_case.epsilon_h * eq_case.B0 * psi_here * np.cos(eq_case.m_h * theta - eq_case.n_h * phi)
        return [
            R * delta_BR * np.cos(theta) / Bphi,
            1.0/q_here - R * delta_BR * np.sin(theta) / (radius * Bphi),
        ]

    residual = deformed_surface_map_residual(
        surface,
        rhs,
        iota,
        alpha_values=np.linspace(0.0, 2*np.pi, n_alpha, endpoint=False),
        state_to_cartesian=lambda state, phi: [
            eq_case.R0 + float(state[0])*np.cos(float(state[1])),
            float(state[0])*np.sin(float(state[1])),
        ],
    )
    return residual.max_residual

[MAGNETIC_SPECTRUM_ATLAS] Multi-component \(B^r\) spectrum atlas#

The single-surface heatmap above is useful for checking a dominant RMP row, but production magnetic-topology analysis usually needs a family view. Here we build a divergence-free multi-component perturbation, compute the Nardon-style contravariant radial spectrum \(\tilde b^1_{mn}=\delta B^1/B_0^3\) on a radial stack, and analyze all requested resonant rows together.

The utilities below are intentionally modular. The displayed atlas uses a wider signed Fourier window, 97 radial surfaces, amplitude_scale='sqrt', and white masks for zero/missing rows. The non-log colormap starts from the same white baseline, so near-zero coefficients do not look artificially separated from masked background. Signed axes show the actual Fourier rows in the computed spectrum: (m,n) is conjugate to (-m,-n) for a real field, but (m,n) and (m,-n) are independent opposite-helicity rows unless the perturbation model imposes an extra symmetry. amplitude_scale='log10' remains useful when the goal is strict dynamic-range auditing rather than visual pattern recognition.

  • plot_rational_surface_map composes optional q-profile, low-order rational markers, projected Poincare points, and island-width bars in the (m/n, s) plane.

  • plot_spectrum_heatmap(..., renderer='pcolormesh') is the recommended surface-spectrum view; it can overlay the physical resonant branch m=-q n_F without drawing the opposite-helicity branch.

  • plot_spectrum_bar3d returns an interactive Plotly figure for rotation, zoom, and hover inspection of dominant rows.

  • plot_radial_mode_heatmap(fixed_n=..., resonant_sign=+1) follows all \(m\) rows at fixed Fourier \(n\) and draws the positive-q branch in the negative-\(m\) half-plane, \(m=-nq(s)\).

  • plot_radial_mode_heatmap(fixed_m=..., axis_convention='fourier') follows actual Fourier \(n\) rows and draws the positive-q branch \(n=-m/q(s)\); the q-curve and island-width bars remain independently toggleable.

Yellow vertical bars mark the single-coefficient Nardon half-width estimate at the low-order rational surface; their thickness scales with the resonant coefficient amplitude. They are an optional overlay, not part of the heatmap itself. Only the physically resonant curve is drawn on each radial map. The opposite-helicity branch is not a conjugate diagnostic and is intentionally not mirrored here. If several harmonics share the same reduced rational surface, the true finite-amplitude island width should be computed from the combined resonant Hamiltonian; the per-row bars are diagnostics, not that nonlinear summed width.
