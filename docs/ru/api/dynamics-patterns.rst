Рабочие процессы динамики и средства расширения
===============================================

``pyna.topo`` предоставляет вспомогательные средства рабочих процессов вокруг основных топологических
объектов. Главная пользовательская точка входа - ``TopologyWorkflow``;
низкоуровневые модули Protocol, Adapter, Builder, Bridge и Factory остаются
доступными для зависимых библиотек, которым нужны стабильные точки расширения.

Фасад рабочего процесса
-----------------------

``TopologyWorkflow`` спроектирован для ноутбуков и повседневных скриптов. Он
объединяет построение системы, интегрирование/итерацию, явное повышение и
сечения без добавления нового математического типа объекта.

Протоколы
---------

Структурные протоколы описывают контракты расширения для внешних систем.
Сторонние объекты могут участвовать, реализуя требуемые атрибуты и методы;
наследование от классов pyna необязательно.

Адаптеры
--------

Адаптеры нормализуют массивы, выводы решателей и существующие объекты pyna в
основные геометрические представления. Они не повышают выборочные данные до
инвариантных объектов молча.

Построители
-----------

Построители кодируют явные правила повышения. Например, траектория может быть
повышена до ``Cycle`` только через вызов построителя или адаптера, который может
требовать замкнутые выборки.

Фабрики и реестры
-----------------

Фабрики предоставляют стабильные точки входа для построения систем, геометрии
и отображений Пуанкаре. Реестры явные и защищены от дубликатов, чтобы тесты и
зависимые библиотеки могли изолировать собственные расширения.
