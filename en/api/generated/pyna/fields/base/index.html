pyna.fields.base#

Abstract base classes for the pyna field hierarchy.

Design principles#

  • A field is a function f: Domain → Codomain. - Domain dimension: 1, 2, 3, or 4. - Codomain rank: 0 (scalar), 1 (vector), 2 (rank-2 tensor).

  • Fields carry a properties attribute (FieldProperty flag set) that downstream code can query before performing operations.

  • Concrete classes implement __call__(coords) and domain_dim.

  • Differential operations (grad, div, curl, …) live in diff_ops.py and return new Field instances, propagating properties automatically.
