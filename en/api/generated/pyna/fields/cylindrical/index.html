Concrete cylindrical-coordinate field implementations.

These replace (and are backward-compatible with):

pyna.fields.cylindrical.validate_phi_grid(phi: numpy.ndarray, *, nfp: int = 1, name: str = 'Phi', allow_single: bool = True) → numpy.ndarray[source]#

Validate and return a cylindrical toroidal-angle grid.

The grid is expected to be one-dimensional, finite, and strictly increasing. For nfp > 1 it must describe one field period, because field evaluation wraps query angles modulo 2*pi/nfp.

pyna.fields.cylindrical.close_periodic_phi_grid(phi: numpy.ndarray, *values: numpy.ndarray, period: float | None = None) → tuple[numpy.ndarray, Ellipsis][source]#

Return a phi grid closed by a duplicate first slice.

If period is not supplied, the period is inferred as phi[-1] + dphi for endpoint=False grids. A grid that already spans 2*pi/Nfp and whose endpoint values match the first slice is treated as already closed.

Bases: pyna.fields.base.VectorField3D

Vector field (BR, BZ, BPhi)(R, Z, φ) on a regular cylindrical grid.

Replaces BOTH:

  • pyna.field_data.CylindricalVectorField (VR/VZ/VPhi naming)

  • legacy toroidal coil-field wrappers (eliminated) (BR/BZ/BPhi naming)

VectorFieldCylind is the canonical public name. The old VectorFieldCylind name remains as a compatibility alias.

Parameters:

  • R (1D ndarray) – Grid axes. Phi may be omitted for a fixed-section 2-D field.

  • Z (1D ndarray) – Grid axes. Phi may be omitted for a fixed-section 2-D field.

  • Phi (1D ndarray) – Grid axes. Phi may be omitted for a fixed-section 2-D field.

  • VR (ndarray) – Vector components in canonical R, Z, Phi order.

  • VZ (ndarray) – Vector components in canonical R, Z, Phi order.

  • BR (VPhi or) – Vector components in canonical R, Z, Phi order.

  • BZ (ndarray) – Vector components in canonical R, Z, Phi order.

  • BPhi (ndarray) – Vector components in canonical R, Z, Phi order.

  • field_periods (int)

  • nfp (int, optional) – Alias for field_periods. If nfp > 1, Phi must cover one field period, not a full torus.

  • name (str)

  • units (str)

  • properties (FieldProperty)

classmethod from_callable(func, R, Z, Phi, n_workers=1, name='', units='', **kwargs)[source]#

Build from a callable.

Supports two calling conventions: - New: func(R_arr, Z_arr, Phi_arr) -> (VR, VZ, VPhi) (vectorised) - Legacy: func(np.array([R, Z, phi])) -> (VR, VZ, VPhi) (point-by-point)

The convention is auto-detected by trying the new signature first.
