pyna.toroidal.geometry#

Toroidal component geometry containers.

These classes provide a small, explicit object layer over the legacy (wall_phi, wall_R_all, wall_Z_all) arrays used by field-line tracing. They are intentionally generic: a first wall, target plate, diagnostic window, or other toroidal component can share the same data model.

class pyna.toroidal.geometry.ToroidalComponentSurface[source]#

R/Z surface sections sampled on a toroidal phi grid.

Parameters:

  • phi – One-dimensional toroidal grid. For nfp > 1 this grid describes one field period, not the full torus.

  • R – Arrays with shape (n_phi, n_poloidal).

  • Z – Arrays with shape (n_phi, n_poloidal).

  • nfp – Number of field periods represented by symmetry.

  • kind – Semantic component type such as "wall", "target_plate", or "glass_window".
